# Agent 工作指令

個人工作偏好。專案有自己的 AGENTS.md 或既有 code 慣例時,以專案為準。

## 語言

- 對話、註解、UI 文案:正體中文(台灣用語),禁簡體字
- 變數名、函式名、commit message:英文
- 對話不中英混用

## 輸出紀律

- 直接講重點,刪填充開場(「值得注意的是」「為了達成這個目標」)
- 禁破折號(全形與半形),改用冒號、括號、逗號、句號
- 不寫聽起來像金句的句子,不做三段式排比
- 就事論事:陳述事實加標準語彙,不自造分類詞、不包裝成 framework

## 禁 emoji 與裝飾符號

- 程式碼、註解、commit、對話、UI 全面禁 emoji、勾選符號、占星符號、任何裝飾性 Unicode
- 箭頭(← ↑ → ↓)作流程、對照、mapping 用途時豁免

## 程式碼風格

個人預設,專案有規範時以專案為準。

- 2 空格縮排、單引號、不加分號
- 復用優先:讀既有檔再開發,共用邏輯抽到共用層
- 單檔超過 500 行考慮拆分,拆前先確認

## 不做假容錯

- 自家服務、DB、API 不可用時回明確錯誤,不回假資料
- catch 區塊不可靜默吞錯後回空結果

## Refactor 安全

- 刪除或內聯函式前,grep 所有引用建對照表,逐項改完做實際呼叫測試
- 編譯通過不等於正確:runtime 錯誤要實際執行驗證
- 本地測試走真實呼叫鏈(router 到 endpoint 到 service),結果跟線上比對

## Commit

- 類型:feat / fix / refactor / docs / chore / test / style / perf / ci / build
- Subject 純英文,禁 CJK
- 訊息就事論事,不誇大、不寫行銷詞
